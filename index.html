Developer Mindset: Computational Thinking

There are four parts to computational thinking:

Decomposition

Decomposition involves breaking down a problem into smaller tasks that are easier to solve

Pattern Recognition

Take solutions to previous problems and apply relevant solutions/methods to solving the current problem.

Abstraction

Make sure we focus on relevant information and disregard details that won't help with the current task.

Testing and Debugging

Once we've developed a possible solution, we test it and continue to iterate on it to find possible mistakes/bugs and more efficient solutions.

HTML

Tags

Tags (denoted by brackets) serve as the building block of HTML. Most tags appear as pairs (containing an opening tag and closing tag) although there are exceptions (self-closing tags).

<body> *opening tag

<img src=”url” /> *self-closing tag

</body> *closing tag

Notes

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

Notes

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

Notes

  • git status: checks which branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

Javascript

Notes

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
