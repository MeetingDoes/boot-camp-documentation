Computational Thinking

There are four parts to computational thinking:

Decomposition

Decomposition involves breaking down a problem into smaller tasks that are easier to solve

Pattern Recognition

Take solutions to previous problems and apply relevant solutions/methods to solving the current problem.

Abstraction

Make sure we focus on relevant information and disregard details that won't help with the current task.

Testing and Debugging

Once we've developed a possible solution, we test it and continue to iterate on it to find possible mistakes/bugs and more efficient solutions.

Agile Methodology

Agile is a set of frameworks and practices for approaching the build and delivery of a product. Agile has a focus on adaptability, collaboration, and self-organization.

Key Concepts

User Stories

Work is broken down into incremental parts. Physically these might be an index card or sticky note with a brief description of the intended functionality. They serve as the basis for “incremental software delivery”.

Incremental Development

An approach to software development in which projects are built while maintaining functionality with each iteration that is “user-visible”.

Daily Meeting

Team convenes each day to bring everyone up to date on information crucial to coordination. Each member briefly describes “completed” contributions and obstacles.

Iterative Development

An approach to software development in which while adding new features/products, we also revisit older features, possibly refactoring/rebuilding them.

Milestone Retrospective

After a project has been active for some time, permanent members of the teams take a few days to create a detailed report of the project's progress/significant events so far.

Personas

In situations where user experience is a major factor, personas (made up users) are used to maintain an anchor/focus for design choices. Personas provide a distinct user expectation as to avoid vague design choices for “ease of use” or “the user” that would ultimately compromise the integrity of the product.

Agile Manifesto

Individuals and interactions over processes and tools
Working software over comprehensive documentation
Customer collaboration over contract negotiation
Responding to change over following a plan

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks which branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
